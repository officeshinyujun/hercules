## 문항 유형 가이드
| 유형 | 설명 | 사용처 |
|------|------|--------|
| single_selection | 5지 선다형. 하나의 정답, 4개 오답 | 개념 정의, 원리 적용, 사실 확인 |
| combination_judgment | 보기(ㄱ~ㄹ) + 선택지(①~⑤) 조합형. 각 보기의 진위를 판단하고 조합을 고름 | 복합 개념 판별, 다중 조건 |
| direct_statement | 자료(표/그래프/보고서)를 직접 보고 판단하는 유형 | 자료 해석, 수치 분석 |
| blank_workflow | 순서도/공정도에서 빈칸 채우기 또는 단계 순서 판단 | 절차, 순서, 프로세스 |

## TPL (Template) — 자료 형식 선택 가이드
각 문항의 stimulus(지문)에 적합한 TPL을 metadata.template에 지정하세요:
| TPL | 적합한 상황 |
|-----|-----------|
| TPL_COMPARATIVE_MATRIX | 두 개 이상의 개념/대상을 속성별로 비교하는 표가 필요할 때 |
| TPL_FORMAL_DOCUMENT | 법령, 공고문, 보고서, 기사 형식의 문서를 제시할 때 |
| TPL_CONVERSATIONAL_FLOW | 두 명 이상의 인물이 대화/인터뷰/토론하는 장면을 제시할 때 |
| TPL_CASE_DIAGNOSTIC_FRAME | 특정 인물의 사례/상황을 제시하고 분석할 때 |
| TPL_SEQUENTIAL_WORKFLOW | 절차/단계/순서를 순서도 형태로 제시할 때 |
| TPL_INSTRUCTIONAL_SCENE | 교수자-학습자 간 수업/발표 장면을 제시할 때 |
| TPL_DIGITAL_FORUM_INTERFACE | 게시글-댓글, SNS, 채팅 기록 형태가 적합할 때 |
| TPL_QUANTITATIVE_CHART | 수치 데이터가 포함된 표를 제시할 때 |
| TPL_PROMOTIONAL_CANVAS | 광고, 포스터, 슬로건, 홍보 자료 형태가 적합할 때 |
